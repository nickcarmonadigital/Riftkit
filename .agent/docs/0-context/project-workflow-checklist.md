# Project Workflow Checklist

**Purpose**: Standard workflow for client projects from intake to handoff.

---

## Project Phases

```
INTAKE → DISCOVERY → PROPOSAL → BUILD → REVIEW → LAUNCH → HANDOFF
```

---

## Phase 1: INTAKE (Day 0)

| # | Task | Done |
|---|------|------|
| 1.1 | Initial contact received | ☐ |
| 1.2 | Qualify lead (budget, timeline, fit) | ☐ |
| 1.3 | Schedule discovery call | ☐ |
| 1.4 | Send calendar invite | ☐ |
| 1.5 | Add to CRM/tracker | ☐ |

---

## Phase 2: DISCOVERY (Day 1-2)

| # | Task | Done |
|---|------|------|
| 2.1 | Complete discovery call | ☐ |
| 2.2 | Fill out discovery template | ☐ |
| 2.3 | Review competitor websites | ☐ |
| 2.4 | Identify industry-specific needs | ☐ |
| 2.5 | Determine deliverables scope | ☐ |
| 2.6 | Send follow-up summary email | ☐ |

---

## Phase 3: PROPOSAL (Day 2-3)

| # | Task | Done |
|---|------|------|
| 3.1 | Create proposal document | ☐ |
| 3.2 | Define deliverables clearly | ☐ |
| 3.3 | Set timeline with milestones | ☐ |
| 3.4 | Calculate pricing | ☐ |
| 3.5 | Send proposal to client | ☐ |
| 3.6 | Follow up if no response in 48hrs | ☐ |
| 3.7 | Handle objections/negotiations | ☐ |
| 3.8 | Get signed agreement | ☐ |
| 3.9 | Collect deposit (if applicable) | ☐ |

---

## Phase 4: BUILD (Days 4-10)

### 4A: Setup

| # | Task | Done |
|---|------|------|
| 4.1 | Create project folder | ☐ |
| 4.2 | Set up development environment | ☐ |
| 4.3 | Collect brand assets (logo, colors, photos) | ☐ |
| 4.4 | Get access to any existing accounts | ☐ |

### 4B: Design

| # | Task | Done |
|---|------|------|
| 4.5 | Choose design archetype | ☐ |
| 4.6 | Select typography | ☐ |
| 4.7 | Define color palette | ☐ |
| 4.8 | Create homepage wireframe/mockup | ☐ |
| 4.9 | Get client approval on design direction | ☐ |

### 4C: Development

| # | Task | Done |
|---|------|------|
| 4.10 | Build homepage | ☐ |
| 4.11 | Build additional pages | ☐ |
| 4.12 | Implement contact form | ☐ |
| 4.13 | Implement booking (if applicable) | ☐ |
| 4.14 | Set up integrations | ☐ |
| 4.15 | Mobile responsive testing | ☐ |
| 4.16 | Cross-browser testing | ☐ |

---

## Phase 5: REVIEW (Days 11-12)

| # | Task | Done |
|---|------|------|
| 5.1 | Internal QA check | ☐ |
| 5.2 | Send preview link to client | ☐ |
| 5.3 | Client review meeting | ☐ |
| 5.4 | Collect feedback (limit to 2 revision rounds) | ☐ |
| 5.5 | Implement revisions | ☐ |
| 5.6 | Get final approval | ☐ |

---

## Phase 6: LAUNCH (Days 13-14)

| # | Task | Done |
|---|------|------|
| 6.1 | Connect domain | ☐ |
| 6.2 | SSL certificate active | ☐ |
| 6.3 | Deploy to production | ☐ |
| 6.4 | Test all links and forms | ☐ |
| 6.5 | Set up analytics | ☐ |
| 6.6 | Submit to Google Search Console | ☐ |
| 6.7 | Take "after" screenshots | ☐ |
| 6.8 | Announce launch to client | ☐ |

---

## Phase 7: HANDOFF (Day 14+)

| # | Task | Done |
|---|------|------|
| 7.1 | Create handoff document (logins, how-tos) | ☐ |
| 7.2 | Quick training session (recorded) | ☐ |
| 7.3 | Collect final payment | ☐ |
| 7.4 | Request testimonial | ☐ |
| 7.5 | Request Google review | ☐ |
| 7.6 | Take case study screenshots | ☐ |
| 7.7 | Schedule 30-day check-in | ☐ |
| 7.8 | Propose retainer (if applicable) | ☐ |
