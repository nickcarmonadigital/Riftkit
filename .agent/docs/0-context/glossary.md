# AI Development Glossary

Quick reference for terms used throughout this framework.

---

## A

**Agent Mode** — AI operating autonomously with file access and tool usage, requiring minimal human intervention.

**Artifact** — A file created during development (code, docs, configs) that has lasting value beyond the conversation.

---

## B

**Brain Dump** — Unstructured feature ideas captured for later processing into structured specs.

**Bun** — Fast JavaScript runtime and package manager, alternative to Node.js + npm.

---

## C

**Claude Code** — Anthropic's AI coding assistant with terminal and file access.

**Context Layer** — Structured information provided to AI to improve response quality.

**Content Cascade** — Extracting multiple short-form pieces from one long-form content piece.

---

## D

**DRY** — "Don't Repeat Yourself" — avoiding code duplication.

**Discovery Call** — Initial client meeting to understand needs and scope.

---

## E

**Edge Function** — Serverless function running at CDN edge locations for low latency.

**ESLint** — JavaScript/TypeScript linting tool for code quality.

---

## F

**Feature Flag** — Toggle to enable/disable features without deploying.

**First Principles** — Breaking problems down to fundamental truths before building solutions.

---

## G

**Gemini** — Google's AI model family.

**Git Flow** — Branching strategy with main, develop, feature, and release branches.

---

## H

**Handoff** — Transferring context between AI sessions or team members.

**Hook** — React pattern for reusable stateful logic.

---

## I

**Implementation Plan** — Detailed technical specification before coding begins.

---

## J

**JSON-LD** — Structured data format for SEO and machine readability.

**JWT** — JSON Web Token for authentication.

---

## L

**LLM** — Large Language Model (GPT, Claude, Gemini, etc.).

**Linting** — Automated code quality checking.

---

## M

**MCP** — Model Context Protocol — standard for AI tool integration.

**Middleware** — Code that runs between request and response.

**Monorepo** — Single repository containing multiple projects/packages.

---

## N

**Next.js** — React framework with SSR, routing, and API support.

**Node.js** — JavaScript runtime for server-side development.

---

## O

**ORM** — Object-Relational Mapping (e.g., Prisma, Drizzle).

---

## P

**Prisma** — Type-safe database ORM for Node.js/TypeScript.

**Project Context** — Living document tracking current project state for AI sessions.

**Prompt Engineering** — Crafting effective instructions for AI systems.

---

## R

**RLS** — Row Level Security — database-level access control (Supabase).

**RSC** — React Server Components.

---

## S

**SOP** — Standard Operating Procedure — documented process.

**SSoT** — Single Source of Truth — authoritative data/doc location.

**SSR** — Server-Side Rendering.

**Supabase** — Open-source Firebase alternative (Postgres + Auth + Storage).

---

## T

**TDD** — Test-Driven Development — write tests before code.

**Tree-shaking** — Removing unused code during bundling.

**TypeScript** — JavaScript with static type checking.

---

## V

**Vite** — Fast frontend build tool.

**Vercel** — Deployment platform for frontend apps.

---

## W

**Walkthrough** — Post-implementation documentation showing what was built and how.

**WI** — Work Instruction — detailed step-by-step task guide supporting SOPs.

---

## Z

**Zod** — TypeScript-first schema validation library.

---

*Terms commonly used in AI-assisted development workflows.*
