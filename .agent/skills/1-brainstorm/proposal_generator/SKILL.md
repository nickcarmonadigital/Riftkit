# Proposal Generator Skill

> **PURPOSE**: Create professional proposals in under 30 minutes.

## 🎯 When to Use

- After completing discovery call
- Client needs a quote/proposal
- Formalizing a project agreement

---

## ⚡ Quick Proposal Workflow

### Step 1: Gather From Discovery (5 min)

Pull from your discovery notes:

```
CLIENT: ________________
MAIN NEED: ________________
SUCCESS METRIC: ________________
TIMELINE: ________________
BUDGET RANGE: ________________
```

### Step 2: Choose Package (2 min)

| Package | Best For | Price Range |
|---------|----------|-------------|
| **Starter** | Simple 1-3 page deliverable | $1,000-1,500 |
| **Standard** | Medium complexity | $2,000-3,000 |
| **Premium** | Full build + integrations | $3,500-5,000 |

### Step 3: Fill Template (15 min)

Fill in:

1. **Client Details** - Name, business, date
2. **Challenge Section** - Their pain points (use their words)
3. **Solution Section** - What you'll deliver
4. **What's Included** - Specific deliverables
5. **Timeline** - Week-by-week breakdown
6. **Investment** - 1-3 pricing options
7. **Why You** - Social proof, differentiator

### Step 4: Review & Send (5 min)

```
[ ] Client name spelled correctly?
[ ] Pricing adds up?
[ ] Timeline realistic?
[ ] CTA is clear ("Reply 'Let's do it'")?
[ ] Valid for 14 days noted?
[ ] Send via email
```

## 💰 Pricing Formulas

### Simple Calculation

```
BASE PRICE = Complexity × Rate
+ Integrations × $200 each
+ Urgency Factor (1.0-1.5x)
= TOTAL
```

### Example

```
Medium complexity × $2,000 = $2,000
2 integrations × $200 = $400
Standard timeline (1.0x)
= $2,400 → Round to $2,500
```

## 📝 Proposal Sections Explained

### The Challenge (1-3 bullets)

Use the client's exact words from discovery:

✅ Good: "You mentioned that customers can't find your availability online"
❌ Bad: "Your digital presence lacks optimization"

### The Solution (Feature List)

Be specific, not vague:

✅ Good: "5-page website with online booking integration"
❌ Bad: "Complete digital transformation solution"

### Timeline

Always pad by 20%:

- If you think 1 week → Say 1.5 weeks
- If you think 2 weeks → Say 2.5-3 weeks

### Pricing Options

Always offer 2-3 options:

| Option | Psychology |
|--------|------------|
| **Option A** (Most Popular) | The one you want them to pick |
| **Option B** (Premium) | Makes Option A look reasonable |
| **Option C** (Budget) | Only if they need lower entry |

## 📄 Proposal Template

```markdown
# Project Proposal: [PROJECT NAME]

**For**: [CLIENT NAME]
**Date**: [DATE]
**Valid**: 14 days

---

## The Challenge

You shared that:
• [Pain point 1 in their words]
• [Pain point 2]
• [Pain point 3]

---

## The Solution

I'll deliver:

✅ [Deliverable 1]
✅ [Deliverable 2]
✅ [Deliverable 3]

---

## What's Included

| Deliverable | Description |
|-------------|-------------|
| [Item 1] | [Details] |
| [Item 2] | [Details] |
| [Item 3] | [Details] |

---

## Timeline

| Week | Activity |
|------|----------|
| Week 1 | Discovery + Planning |
| Week 2 | Build + Development |
| Week 3 | Review + Revisions |
| Week 4 | Launch + Handoff |

---

## Investment

### Option A: [Package Name] (Recommended)
**$X,XXX**
Includes [summary of what's included]

### Option B: [Premium Package]
**$X,XXX**
Includes everything in Option A, plus [extras]

---

## Next Steps

1. Reply "Let's do it" to accept
2. I'll send an invoice for 50% deposit
3. We'll schedule our kickoff call

---

Questions? Just reply to this email.

[YOUR NAME]
```

## ⚠️ Common Mistakes

| Mistake | Fix |
|---------|-----|
| Pricing too low | Use formula, not gut |
| Vague deliverables | Be stupidly specific |
| No deadline | "Valid for 14 days" |
| Complex payment terms | Keep simple: 50/50 or 100% upfront |
| No next steps | End with clear CTA |
