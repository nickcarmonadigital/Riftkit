# Change Management Skill

**Purpose**: Establish an auditable trail for every production change by automatically generating change records as GitHub Issues, maintaining a monthly change log, and providing emergency change procedures. This satisfies SOC2 CC8.1 control requirements without introducing heavyweight ITSM tooling, keeping everything within GitHub.

## TRIGGER COMMANDS

```text
"Change management records"
"SOC2 change log"
"Deployment audit trail"
"Set up change management"
"Using change_management skill: configure for [project]"
```

## When to Use
- When preparing for SOC2 Type II audit and need change management evidence
- When establishing production deployment governance for the first time
- When compliance requires documented approval and audit trails for changes
- When the team needs visibility into what changed, when, and why

## PROCESS

### Step 1: Define Change Record Structure

Every production deployment creates a change record containing:

| Field | Source | Example |
|-------|--------|---------|
| Change ID | Auto-generated | CHG-2026-0142 |
| Timestamp | Workflow run time | 2026-02-25T14:30:00Z |
| Type | Label on PR | `change:standard` or `change:emergency` |
| Description | PR title + body summary | "Add rate limiting to /api/v1/upload" |
| Deployed SHA | Git SHA | abc123def456 |
| Approved By | PR reviewers | @team-lead |
| Rollback Plan | PR template field | "Revert commit abc123" |
| Verification | Post-deploy result | PASS |

### Step 2: Automated Change Record Creation

```yaml
# .github/workflows/change-record.yml
name: Create Change Record

on:
  workflow_run:
    workflows: ["Gated Production Deploy"]
    types: [completed]

permissions:
  issues: write
  contents: read

jobs:
  create-record:
    if: github.event.workflow_run.conclusion == 'success'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          ref: ${{ github.event.workflow_run.head_sha }}

      - name: Gather change metadata
        id: meta
        run: |
          SHA="${{ github.event.workflow_run.head_sha }}"
          SHORT_SHA="${SHA:0:7}"
          YEAR=$(date -u +%Y)
          MONTH=$(date -u +%m)
          TIMESTAMP=$(date -u +%Y-%m-%dT%H:%M:%SZ)

          # Get PR info for this SHA
          PR_DATA=$(gh api repos/${{ github.repository }}/commits/$SHA/pulls --jq '.[0]')
          PR_TITLE=$(echo "$PR_DATA" | jq -r '.title // "Direct push"')
          PR_NUMBER=$(echo "$PR_DATA" | jq -r '.number // "N/A"')
          PR_AUTHOR=$(echo "$PR_DATA" | jq -r '.user.login // "${{ github.actor }}"')
          REVIEWERS=$(echo "$PR_DATA" | jq -r '[.requested_reviewers[]?.login] | join(", ") // "N/A"')

          # Get commit message
          COMMIT_MSG=$(git log -1 --format="%B" "$SHA" | head -5)

          # Generate change ID
          COUNT=$(gh issue list --label "change-record" --search "CHG-$YEAR" --state all --json number --jq 'length')
          CHANGE_ID="CHG-$YEAR-$(printf '%04d' $((COUNT + 1)))"

          echo "change_id=$CHANGE_ID" >> "$GITHUB_OUTPUT"
          echo "sha=$SHA" >> "$GITHUB_OUTPUT"
          echo "short_sha=$SHORT_SHA" >> "$GITHUB_OUTPUT"
          echo "timestamp=$TIMESTAMP" >> "$GITHUB_OUTPUT"
          echo "pr_title=$PR_TITLE" >> "$GITHUB_OUTPUT"
          echo "pr_number=$PR_NUMBER" >> "$GITHUB_OUTPUT"
          echo "pr_author=$PR_AUTHOR" >> "$GITHUB_OUTPUT"
          echo "reviewers=$REVIEWERS" >> "$GITHUB_OUTPUT"
        env:
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}

      - name: Create change record issue
        run: |
          gh issue create \
            --title "${{ steps.meta.outputs.change_id }}: ${{ steps.meta.outputs.pr_title }}" \
            --label "change-record" \
            --body "## Change Record

          | Field | Value |
          |-------|-------|
          | **Change ID** | ${{ steps.meta.outputs.change_id }} |
          | **Timestamp** | ${{ steps.meta.outputs.timestamp }} |
          | **Type** | Standard |
          | **SHA** | \`${{ steps.meta.outputs.sha }}\` |
          | **PR** | #${{ steps.meta.outputs.pr_number }} |
          | **Author** | @${{ steps.meta.outputs.pr_author }} |
          | **Reviewers** | ${{ steps.meta.outputs.reviewers }} |
          | **Verification** | Passed (triggered by successful deploy) |

          ### Description
          ${{ steps.meta.outputs.pr_title }}

          ### Rollback Plan
          \`\`\`bash
          git revert ${{ steps.meta.outputs.short_sha }} --no-edit && git push origin main
          \`\`\`

          ---
          *Auto-generated by change management workflow*"
        env:
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

### Step 3: Emergency Change Procedure

Create a template for emergency changes that bypass normal approval:

```markdown
<!-- .github/ISSUE_TEMPLATE/emergency-change.md -->
---
name: Emergency Change Request
about: Document an emergency production change
title: "EMERGENCY CHG: "
labels: change-record, emergency-change
---

## Emergency Change Record

| Field | Value |
|-------|-------|
| **Requestor** | @<!-- your handle --> |
| **Severity** | P0 / P1 |
| **Incident** | #<!-- link to incident issue --> |
| **Time of Change** | <!-- UTC timestamp --> |

### What changed
<!-- Describe the exact change made -->

### Why it was emergency
<!-- Why couldn't this wait for normal change process? -->

### Rollback plan
<!-- How to undo this change if needed -->

### Post-change verification
- [ ] Health checks passing
- [ ] Error rate returned to baseline
- [ ] Affected users notified

### Post-incident review
- [ ] Scheduled within 48 hours
- [ ] Root cause identified
- [ ] Prevention plan documented
```

### Step 4: Monthly Change Log Generation

```yaml
# .github/workflows/monthly-change-log.yml
name: Monthly Change Log

on:
  schedule:
    - cron: '0 6 1 * *'  # First day of each month at 6:00 UTC

permissions:
  issues: read
  contents: write

jobs:
  generate-log:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Generate monthly change log
        run: |
          LAST_MONTH=$(date -u -d "1 month ago" +%Y-%m)
          YEAR=$(echo "$LAST_MONTH" | cut -d- -f1)
          MONTH=$(echo "$LAST_MONTH" | cut -d- -f2)

          echo "# Change Log: $LAST_MONTH" > "docs/change-logs/$LAST_MONTH.md"
          echo "" >> "docs/change-logs/$LAST_MONTH.md"
          echo "| Change ID | Date | Type | Description | Author |" >> "docs/change-logs/$LAST_MONTH.md"
          echo "|-----------|------|------|-------------|--------|" >> "docs/change-logs/$LAST_MONTH.md"

          gh issue list \
            --label "change-record" \
            --state all \
            --search "CHG-$YEAR" \
            --json title,createdAt,labels,assignees \
            --jq '.[] | select(.createdAt | startswith("'"$LAST_MONTH"'"))' \
            | jq -r '"\(.title) | \(.createdAt[:10]) | \([.labels[].name] | if any(. == "emergency-change") then "Emergency" else "Standard" end) | \(.assignees[0].login // "N/A")"' \
            | while IFS='|' read -r title date type author; do
                echo "| $title | $date | $type | $author |"
              done >> "docs/change-logs/$LAST_MONTH.md"

          echo "" >> "docs/change-logs/$LAST_MONTH.md"
          echo "---" >> "docs/change-logs/$LAST_MONTH.md"

          TOTAL=$(gh issue list --label "change-record" --state all --search "CHG-$YEAR created:$LAST_MONTH-01..$YEAR-$MONTH-01" --json number --jq 'length')
          EMERGENCY=$(gh issue list --label "emergency-change" --state all --search "created:$LAST_MONTH-01..$YEAR-$MONTH-01" --json number --jq 'length')
          echo "**Total changes**: $TOTAL | **Emergency changes**: $EMERGENCY" >> "docs/change-logs/$LAST_MONTH.md"
        env:
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}

      - name: Commit change log
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git add docs/change-logs/
          git commit -m "docs: add $(date -u -d '1 month ago' +%Y-%m) change log" || exit 0
          git push
```

### Step 5: SOC2 CC8.1 Control Mapping

| SOC2 CC8.1 Requirement | How This Skill Satisfies It |
|-------------------------|----------------------------|
| Changes are authorized | GitHub PR reviews + environment approvals |
| Changes are documented | Automated change record issues |
| Changes are tested | Pre-deploy gate requires passing CI |
| Emergency changes are documented | Emergency change issue template |
| Changes are traceable | SHA-linked records with PR references |
| Change log is maintained | Monthly automated change log generation |
