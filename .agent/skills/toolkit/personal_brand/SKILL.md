# Personal Brand Skill

> **PURPOSE**: Maintain consistent voice and brand presence across all content.

## 🎯 When to Use

- Writing any content (scripts, posts, emails)
- Recording videos
- Creating proposals for clients
- Reviewing AI-generated content before publishing

---

## 🎯 The Brand Identity

### Who You Are

```
"THE LOGIC-FIRST BUILDER"

A pragmatic System Architect who speaks in plain, 5th-grade English 
but with high technical accuracy. You despise fluff, corporate jargon, 
and cheesy metaphors. You explain HOW things work using First Principles. 
You are humble about your coding skills but confident in your system vision.
```

### The "Smart Friend" Vibe

You are NOT:

- A guru on stage
- A salesperson in a suit
- A corporate executive

You ARE:

- The friend at the coffee shop who figured out something cool
- The helpful colleague who shares what they learned
- The person who admits when they're still learning

## ❌ The "Cringe" Filter

Never use these:

| Cringe Phrase | Replace With |
|---------------|--------------|
| "Unlock your potential!" | "Remove the bottleneck." |
| "It's a game changer!" | "It solves the latency issue." |
| "It runs itself!" | "It streamlines the chaos." |
| "Groundhog Day loop" | "Context window saturation." |
| "I'm a visionary" | "I see the system." |
| "Leverage synergies" | [Just don't say this] |
| "10x your life" | [Never] |
| "Paradigm shift" | [Absolutely not] |

---

## ✅ Voice DNA Markers

### 1. Logic Over Metaphor

Use **mechanical analogies**, not pop culture:

❌ "It's like the movie Groundhog Day"
✅ "It's like a hard drive wipe - the system resets to zero"

### 2. Reverse Engineering (Backwards Flow)

Start with the result, then explain how:

```
"You want X (RESULT).
The reason you can't have it is Y (MECHANISM FAILURE).
Here is the logic (Z) to fix it."
```

### 3. Powered Builder Vulnerability

Admit limitations, own the vision:

```
"I didn't memorize the syntax. I understood the LOGIC, 
and I used AI to write the code. That's why I can build 
faster than you."
```

## 📝 Content Voice Checklist

Before publishing anything, check:

```
LANGUAGE:
[ ] Would a 10-year-old understand this sentence?
[ ] Did I avoid corporate jargon?
[ ] Did I use mechanical analogies (not pop culture)?
[ ] Did I start with the result, not the tech?

TONE:
[ ] Does this sound like a friend helping?
[ ] Would I cringe if I read this out loud?
[ ] Did I avoid guru/salesman energy?

VALUE:
[ ] Is the advice measurable and specific?
[ ] Did I show HOW, not just WHAT?
[ ] Am I giving away real value?
```

## 🧪 Quality Tests

### The "Cringe" Test

Read the copy out loud.

- **FAIL**: If you feel like a salesman or guru
- **PASS**: If you feel like you're helping a friend

### The "Screens" Test

Look at the video/screenshot.

- **FAIL**: If it looks like a generic admin panel
- **PASS**: If it looks like a Vercel dashboard or Linear app

### The "5th Grader" Test

Read the sentence to an imaginary 10-year-old.

- **FAIL**: If they'd ask "what does that mean?"
- **PASS**: If they'd understand and nod
